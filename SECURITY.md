# Security Policy

## Supported Versions

Security fixes are generally applied to the latest released version.

## Reporting a Vulnerability

Please do not open public issues for security vulnerabilities.

Instead:

1. Use GitHub Security Advisories in this repository for private reporting.
2. If advisories are unavailable, contact: manish.amin.security@proton.me
3. Include detailed reproduction steps.
4. Include impact and affected versions if known.

## What to Include

- Package version
- Runtime environment
- Clear reproduction steps
- Expected behavior and actual behavior
- Any proof-of-concept details

## Response Targets

- Initial acknowledgment: within 3 business days
- Triage update: within 7 business days
- Resolution timeline: depends on severity and complexity

## Disclosure Process

After remediation is available, maintainers will coordinate responsible disclosure and publish release notes.
